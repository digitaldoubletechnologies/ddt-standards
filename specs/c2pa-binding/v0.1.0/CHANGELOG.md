# Changelog — C2PA Binding Specification

This changelog records **standards-track** revisions to the DDT C2PA Assertion Binding Specification.

This document does not imply implementation readiness, enforcement, or platform obligations.

---

## v0.1.0 — Initial publication

**Status:** Draft for standards discussion  
**Date:** 2026-02-04

### Added
- Initial definition of the `ddt.cro_binding` payload
- Reference-only binding model (CRO ID + receipt hash)
- Explicit non-enforcement posture aligned with DDT-000
- Example payloads covering active, revoked, expired, and superseded consent states

### Notes
- This version is intended for **discussion and feedback** with standards bodies and platform stakeholders.
- Future revisions will be additive and backwards-compatible where possible.
